Problem Solving

Stopped by a problem

One problem I was stuck by was the difference in calling functions on different data types. Specifically I was trying to call myObject.values() much like you might call myArray.push(someVar). When I should have been using Object.values(myObject). To help solve the issue I read the error message which was:

"TypeError: object.values is not a function".

Which was even more confusing because I was sure it was a function. So instead I started googling, and ended up reading the mdn page for Object.values(), and where I experimented with the Javascript demo on the page, and I managed to figure out the issue I was having. Initially I felt frustrated and confused when first starting out on this problem, but as I started to work things out I felt... not quite statisfied, but maybe relief that I had figured out this problem. Although I definitely felt good learning about object prototypes.

Solving a problem

A problem I enjoyed solved was the best way to assign the students test scores array to each of the student objects stored in the gradebook object. I wasn't sure immediately how to solve this issue. Previously I searched up good ways to loop through an array, and read a stack overflow talking about use cases for loops, which was very helpful. So I went with the forEach loop. I wanted to use the index of the students array to get the key in the gradebook but had some trouble creating a testScores key for the student object. I was trying something like:

scores.forEach( (score, index) => (gradebook.students.index.testScores = score))

And I was getting the error message:

TypeError: can't access property "index", gradebook.students is undefined

Which was very confusing because I had thought I had just defined my gradebook.students a line earlier, but now its undefined. So I added a console.log(gradebook), to make sure they still existed, which they did, so I did some googling and found some things about dot and bracket notation that helped figure out I needed to use the bracket notation to access the students correctly.

  • Reading error messages: I am pretty somewhat confident in reading error messages, in python. But less experienced reading them in JavaScript.
  • Googling: I am very confident in google issues I am having.
  • Console.logging: Definitely a favourite of mine to problem solve.
